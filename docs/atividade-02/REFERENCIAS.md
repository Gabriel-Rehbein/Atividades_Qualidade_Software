# Qualidade do Processo – Startup Local Eats

## 1. Estrutura de Gestão de QA

A qualidade não deve ser uma etapa final, mas sim integrada ao processo de desenvolvimento.

### Princípio Central
> Qualidade é responsabilidade de todos

Aplicação:
- PO → define requisitos claros
- Dev → garante qualidade técnica
- QA → valida experiência e riscos
- DevOps → garante ambiente confiável

---

## 2. Organização de Papéis (Base: Gestão de Projetos)

A ausência de papéis bem definidos gera:
- Retrabalho
- Falhas em produção
- Problemas como pedidos duplicados

### Estrutura recomendada

- **Product Owner (PO):**
  - Define backlog
  - Cria critérios de aceite

- **Desenvolvedor:**
  - Implementa funcionalidades
  - Cria testes unitários
  - Participa de code review

- **QA:**
  - Atua desde o início
  - Cria estratégia de testes
  - Identifica riscos antes do código

- **DevOps:**
  - Garante ambientes iguais (dev/test/prod)
  - Automatiza deploy

---

## 3. Cultura de QA (Agile Testing)

QA não é apenas testar no final.

### Novo papel do QA:
- Facilitador da qualidade
- Preventivo (antes do erro acontecer)
- Integrado ao time

### Práticas:
- QA participa das refinamentos
- QA revisa requisitos
- QA ajuda a escrever critérios de aceite

---

## 4. Quadrantes de Testes Ágeis

### Organização dos testes

#### Q1 – Testes Técnicos (Base do sistema)
- Testes unitários
- Testes de integração
- Ex: evitar duplicidade no banco

#### Q2 – Testes de Negócio (Funcionalidade)
- Testes funcionais
- Testes de aceitação
- Ex: validar fluxo de pedido

#### Q3 – Testes Exploratórios
- Testes manuais
- Simulação de uso real

#### Q4 – Testes Não Funcionais
- Performance
- Segurança
- Carga

---

## 5. Adoção do Scrum

### Estrutura recomendada

- Sprint (1–2 semanas)
- Daily (15 min)
- Review (entrega)
- Retrospectiva (melhoria contínua)

### Benefícios:
- Feedback rápido
- Menos erros acumulados
- Melhor organização

---

## 6. Padrões Profissionais (QA)

### Base: ISTQB / CTFL

Competências esperadas:
- Escrita de casos de teste
- Reporte de bugs
- Conhecimento em tipos de teste
- Noções de automação

---

## 7. Conclusão

A melhoria da qualidade na Local Eats depende de:

- Definição clara de papéis
- Cultura de responsabilidade compartilhada
- QA atuando desde o início
- Uso de metodologias ágeis (Scrum)
- Estrutura de testes organizada

Sem isso, os problemas atuais continuarão ocorrendo.
