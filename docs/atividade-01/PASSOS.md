

## 🛠️ O que vocês precisam fazer

### 📍 Passo 1 — Compreender o cenário

Leiam atentamente o contexto e os relatos apresentados.

Procurem entender:

- Qual é o propósito do sistema  
- Quem são seus usuários  
- Quais problemas estão sendo enfrentados  

---

### 📍 Passo 2 — Identificar os problemas do produto

Elaborem uma lista clara e objetiva com os problemas percebidos no sistema.

---

### 📍 Passo 3 — Relacionar problemas a atributos de qualidade

Para cada problema identificado, indiquem:

- Qual característica de qualidade do produto está sendo afetada?  
- Utilizem como base os atributos de qualidade estudados em aula (ISO/IEC 25000).  

---

### 📍 Passo 4 — Justificar tecnicamente

Explique de forma breve:

- Por que esse problema afeta esse atributo de qualidade  
- Qual o impacto para o usuário ou para o negócio  

---

## 🔎 Perguntas que ajudam na análise

- O sistema executa corretamente as funções que promete?  
- É fácil de usar e entender?  
- Funciona de forma estável?  
- Apresenta bom desempenho?  
- Funciona adequadamente em diferentes dispositivos e ambientes?  
- Gera frustração ou retrabalho ao usuário?  

---

## 📦 Entrega da atividade

Cada grupo deverá entregar:

- Lista de problemas identificados no produto  
- Atributos de qualidade possivelmente afetados  
- Justificativas técnicas para cada relação identificada  

---

|----------------------------|------------------------------|----------------------|
